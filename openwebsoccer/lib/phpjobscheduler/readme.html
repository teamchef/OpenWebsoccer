phpJobScheduler wurde designed,um Jobs regelmäßig automatiesiert auszuführen.
Diese Version führt dies Jobs im Hintergrund aus,ohne das Ausgaben erfolgen. Diese werden inclusive von Fehlermeldungen in die Datenbank geschrieben.
Genaue Details können sie dort einsehen: www.phpjobscheduler.co.uk

Voraussetzungen:

  • MySQL ( sollte nun auch mit vielen anderen Datenbanken laufen,verändern sie entsprechend die Einstellungen in dieser Datei: in  /phpjobscheduler/pjsfiles/constants.inc.php )
  • PHP 5.x

Die Durchführung eines Upgrades bei einer vorhandenen Installation:  Hierbei sind zwei Hauptpunkte zu beachten !

  1. Wenn Jobs existieren,so werden diese in der Datenbank unverändert so belassen. 
    Sie können auch die Konfigurationsdatei behalten:  /phpjobscheduler/pjsfiles/config.inc.php

    Haben sie ß„nderungen an dieser Datei gemacht :  /phpjobscheduler/pjsfiles/constants.inc.php
    so müssen sie diese ß„nderungen erneut vornehmen,da Veränderungen vom Upgrade vorgenommenn werden. Machen sie daher zuvor eine Kopie dieser Datei.

  2. Haben sie die obigen Anweisungen befolgt,so müssen sie sich einloggen,um die Jobs zu überprüfen,so werden benötigte neue Felder in der Datenbank automatisch korrigiert.


Neuinstallation durchführen:

  1. Entpacken sie das heruntergeladene Verzeichnis.

  2. Editieren sie folgende Datei mittels Notepad bzw. ihren bevorzugten PHP-Editor: /phpjobscheduler/pjsfiles/config.inc.php

  3. Laden sie dann das Verezeicnis auf ihren Server.

  4. Sehr wichtig ! Schützen sie unbedingt das Verzeichnis "pjsfiles" unbedingt mit einem Passwort. - Wenn sie dies nicht tun,k ann jeder,der dieses Verzeichnis erreicht sämtliche Einstellunge der Jobs ändern ! Eine htaccess kann auf allen Systemen eingerichtet werden,damit zuerst eine Zugangskontrolle über Username und Password erfolgt. ( Auf Windows nutzen die das Control Panel. )  ( Bitte kontaktieren sie ihren Anbietern bei Problemen und nicht mich ! ).

  5. Der folgende Punkt muss bei einer Ausführung über Cron nicht gemacht werden,sondern dient als Alternative für Websites ohne Cronsupport.

    Fügen sie folgendes z.B. in die index.php bzw. index.html ein,dadurch werden die Jobs über diese Dateien angestoßen.  Dabei gibt es zwei Methoden:

    1. Am Anfang der PHP Datei:

      <?php include( dirname(__FILE__) . "/phpjobscheduler/firepjs.php"); ?>

    2. oder einfach in die HTML Seite,bzw. jede andere besuchte Seite ihrer Webpräsens:
    Beachte unbedingt,das 'DEINEDOMAIN' auch auf deine Domain zeigt !
    Der obige HTML-Code kann auf jede beliebige HTML-Seite eingefügt werden. Unsichtbar,klein und behindert nicht die Seitenausführung.

     

  6. Bitte hier klicken,um neue Jobs hinzu zu fügen.

    7. Wenn sie korrekt die obigen Schritte befolgt haben,so werden die folgenden Datentabellen in der Datenbank erzeugt:

      phpJobScheduler

  7. Hier klicken um die Jobs zum Test zu starten. Dies erlaubt es die Ausgaben zu überprüfen und ob alles wie gewünscht ausgeführt wird.  Wenn DEBUG gesetzt ist,so wird folgendes ausgegeben: "Found script to run...",etc.



  8. Wenn alles wie gewünscht läuft setzen sie DEBUG auf false,indem sie die Datei: /phpjobscheduler/pjsfiles/config.inc.php entsprechend editieren.

Error log - Jede Jobausführung wird geloggt,was eine Dokumentation der Ausführungszeiten gleichkommt. Darüber hinaus sind evebtuelle Fehler erkennbar. Sofern Die Jobs auch einen Rückgabewert liefern,sind diese ebenso einsehbar. Besonders die Ausführungszeit eines Joba kann wertvolle Aufschlüsse ergeben,ob im ausgeführtem Script die Laufzeit zu lange ist.

Erfasste Daten: Datum und ketze Ausführungszeit mit der URL eventueller Fehlerausgabe. "Ausgabe:" "ohne Daten" bedeutet,das die Ausführung des Scripts ohne Fehler und ohne Rückgabewert erfolgte.

Die Standardeinstellung ist(TRUE),das Logging ist eingeschaltet. Um die Einstellung ein/aus zu schalten,muss man wie folgt vorgehen:

ß„ndern sie die Variable ERROR_LOG in der Datei /pjsfiles/constants.inc.php . FALSE stopt die Aufzeichnung. Es beeinflusst nicht die Ausführung der Jobs.

Ein Logeintrag hat eine maximale Länge von 1200 Zeichen. Dies kann mann mit der Konstante MAX_ERROR_LOG_LENGTH in der Datei constants.inc.php ändern.

 

 

 

Veränderung des Zeitfensters

- Dies gilt für die Ausführung ohne Cronjob !

Das Zeitfenster kann durch eine Anpassung der Variable TIME_WINDOW in der Datei /pjsfiles/constants.inc.php gesetzt werden.  Die Grundeinstellung ist 3600 Sekunden ( 60 Minuten ,dies sollte für die meisten Zwecke reichen ).  Dies bedeutet,das durch das ausführen der Datei phpjobscheduler/firepjs.php) alle Jobs in einem Zeitrahmen von 60 Minuten ausgelöst werden.

Sie können diesen Zeitrahmen nach oben oder untern entsprechend verändern.  Sollte ihre Seite zu wenig Besucher haben,so können sie die Einstellung auf bis zu 43200 ( 12 Stunden ) erhöhen.

Wenn ihre Seite IMMER viele Besucher haben,so reduzieren sie die Einstellung entsprechend.

Wenn sie weitere hilfe benötigen schauen sie ins Forum
